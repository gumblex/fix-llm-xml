# fix_xml_with_text_tags 函数需求
## 一、函数定位
用于修复大语言模型（LLM）输出的结构损坏的XML文本，针对指定的**纯文本标签**（预设为无子标签、内容为任意文本/代码的标签）做专项修复，确保修复后的XML符合标准格式，可被常规XML解析器正常解析，且纯文本标签内的所有原始内容完整保留、不会被误识别为XML结构。

## 二、函数定义
```python
def fix_xml_with_text_tags(xml: str, text_tags: Iterable[str] = ()) -> str:
```
### 参数说明
1. `xml`：待修复的XML字符串，可包含格式错误、未闭合标签、嵌套CDATA、未转义特殊字符等问题
2. `text_tags`：纯文本标签名列表，区分大小写，支持带命名空间前缀（如`ns:code`），列表内的标签将按纯文本规则专项修复
### 返回值
修复完成的合法XML字符串

## 三、通用基础解析规则
对输入XML做全局流式解析时，遵循以下规则：
1. 可识别的XML元素仅包含四类：开始标签、结束标签、自闭合标签、XML注释；所有包裹在`<>`内、无法识别为上述四类的内容，均作为普通字面文本处理
2. 维护全局普通标签栈：
   - 解析到非纯文本类的开始标签时，将标签名压入栈
   - 解析到结束标签时，若标签名与栈顶标签名一致则弹栈，否则直接丢弃该多余结束标签
   - 遍历完全部XML内容后，若栈中仍有未闭合的普通标签，按栈逆序依次补全对应的闭合标签

## 四、纯文本标签专项处理规则
为了正确找到外层标签的结束位置，要在纯文本标签内的内容，解析其内层嵌套关系。内层可多层嵌套，包含内层同名标签（是否CDATA模式）、内部普通CDATA块三种类型。找到结束位置后，提取外层标签内的原始文本（不含外层标签本身及CDATA前后缀）。外层模式决定如何包裹/转义这段原始内容。
当解析到的开始标签名属于`text_tags`列表，或开始标签后紧跟类CDATA前缀时，进入该标签的**纯文本修复模式**，该模式下内部内容默认不识别为普通XML子标签，仅按以下规则处理。

### 4.1 外层属性识别
首先识别纯文本标签内容开头的类CDATA前缀：优先匹配`<![CDATA[`，其次匹配正则`[<\[({!]+CDATA[\[({>]`（忽略大小写），若匹配成功则标记该纯文本标签为**CDATA外层模式**，否则标记为**普通外层模式**。

### 4.2 内部内容处理规则
纯文本模式下维护独立的内部栈，栈元素记录每个层级的类型与属性（外层纯文本标签/内层同名标签/内部普通CDATA块，以及是否为CDATA开头），按优先级从高到低识别内部内容：
1. （仅在当前状态为开启内层/外层含CDATA标签，或普通CDATA块时）优先识别`ESCAPED_CDATA_END`序列（`]]]]><![CDATA[>`），直接作为纯文本保留（要特殊标记，防止后续二次转义）
2. （仅在当前状态为开启内层/外层含CDATA标签时）其次识别`]]></当前纯文本标签名>`格式的CDATA+闭合标签组合
3. （仅在当前状态为开启外层含CDATA标签时）其次识别兼容性正则 `(</CDATA>|\]+>+)` + `</当前纯文本标签名>`格式的CDATA+闭合标签组合
4. （仅在当前状态为开启内层无CDATA标签/外层任意情况时）其次识别`</当前纯文本标签名>`格式的普通闭合标签
5. （仅在当前状态为开启内层普通CDATA块时）其次识别标准CDATA结束标记`]]>`
6. （仅在当前状态为开启内层/外层含CDATA标签，或普通CDATA块时）其次识别标准CDATA开始标记`<![CDATA[`
7. （所有情况）其次识别同名纯文本标签开始标记，并判断其是否为标准的CDATA开头
8. 其余所有内容（非同名标签、孤立的CDATA结束标记、无法识别的`<>`内容等），均作为普通文本保留

### 4.3 闭合匹配规则
（这是 4.2 的具体说明，优先级按照 4.2 处理）
根据内部栈状态匹配对应闭合标记，直到外层纯文本标签的闭合标记被匹配，或遍历到整个XML的末尾：
- 若为CDATA外层模式，优先匹配`]]></当前标签名>`，其次匹配兼容性正则 `(</CDATA>|\]+>+)` + `</当前标签名>`，最后无匹配则匹配`</当前标签名>`
- 若为普通外层模式，匹配`</当前标签名>`

若遍历到XML末尾仍未匹配到外层闭合标记，内部栈未清空，则重新依次从右边搜索 `]]></当前标签名>`（CDATA开始）、`兼容正则</当前标签名>`（CDATA开始）、`</当前标签名>`，作为外层的关闭标签。
若还是未找到，则之后依次补关闭标签。
无论内层元素是否有足够的闭合标记，必须保证外层纯文本标签可正常闭合。完全不补内层闭合标记。

### 4.4 内容转义修复规则
退出纯文本修复模式后，返回 4.1~4.3 捕获的纯文本标签内容（可能是分段token的形式）。返回的纯文本标签内容不含CDATA前后缀（或兼容性正则），为纯文本标签/CDATA内原始文本，不做任何增删。
按外层模式对纯文本内容做转义修复：
1. **CDATA外层模式**：
   - 修复多层嵌套CDATA的合法性：内部嵌套的CDATA结束标记自动转义为`ESCAPED_CDATA_END`，已存在的`ESCAPED_CDATA_END`不做二次转义
2. **普通外层模式**：
   - 所有普通文本内容中的`<`替换为XML转义符`&lt;`，`>`替换为XML转义符`&gt;`，其余内容保持不变
3. 无论哪种模式，纯文本标签内的所有原始内容必须完整保留，仅做XML规范要求的必要转义，解析后可完全还原原始内容

## 五、输出要求
修复后的XML需满足：
1. 标签闭合符合标准XML语法规范，可被 lxml 等宽松XML解析器正常解析（可能存在逻辑错误、处理指令等，不影响解析）
2. 所有非纯文本标签的结构保持与原始输入一致，仅补全缺失闭合标签、删除多余闭合标签
3. 所有纯文本标签的内容完整保留，解析后，正确转义的内容应与原始输入内容完全一致；错误转义的内容不额外添加标签等，没有二次转义
